# ГАЙД ПО МОДУЛЮ М2: Контейнеризация TodoApp

## Пошаговый план реализации

### Шаг 1: Создание Dockerfile

Dockerfile — это инструкция для Docker, как собрать образ вашего приложения. Мы используем **Multi-stage build** (многоэтапную сборку).

**Зачем это нужно?**

1. На этапе сборки нам нужен тяжелый SDK (800МБ+), чтобы скомпилировать код.
2. На этапе запуска нам нужен только легкий Runtime (около 200МБ), чтобы просто запустить готовые файлы.
   Это делает итоговый образ маленьким и безопасным.

**Как писать Dockerfile:**

1. Начните с базового образа SDK: `FROM mcr.microsoft.com/dotnet/sdk:10.0 AS builder`.
2. Скопируйте файлы `.csproj` и восстановите пакеты через `dotnet restore`. Это важно делать ДО копирования всего кода, чтобы Docker кэшировал зависимости и не скачивал их при каждом изменении одной строчки кода.
   Данная команда копирует file из папки src1/ в вашем репозитории в
   папку src2/

```Dockerfile
COPY src1/file src2/
```

3. Скопируйте остальной код и выполните `dotnet publish`.
4. В этом же образе создайте **Migration Bundle** (бандл миграций). Это исполняемый файл, который содержит всё необходимое для обновления базы данных.
Для установки:
```sh
dotnet tool install --global dotnet-ef
```
Для создания bundle под линукс(исполняемый файл, который делает миграции бд)
```sh
dotnet ef migrations bundle --self-contained -r linux-x64 -o /app/efbundle
```

5. Перейдите ко второму этапу `FROM mcr.microsoft.com/dotnet/aspnet:10.0  AS `, скопируйте туда только результат публикации и бандл миграций.

### Шаг 2: Создание Docker Compose

Когда приложению нужна база данных, управлять двумя контейнерами вручную неудобно. Для этого есть `docker-compose.yml`.

**Ключевые блоки:**

- **services**: Список ваших контейнеров. У нас это `db` (PostgreSQL) и `backend`.
- **healthcheck**: У базы данных есть проверка здоровья. Бэкенд не должен запускаться, пока база не скажет "я готова принимать соединения".
- **environment**: Здесь мы прокидываем настройки. Обратите внимание, что в строке подключения вместо `localhost` мы пишем имя сервиса базы — `db`. Docker сам поймет, куда подключаться.
- **volumes**: Тома нужны, чтобы данные базы данных не удалялись при перезапуске контейнера.

### Шаг 3: Скрипт запуска (entrypoint)

В контейнере нам нужно сделать две вещи: обновить базу данных и запустить приложение. Мы создаем `scripts/apply-migrations.sh`, который:

1. Запускает `./EfCoreMigrationsBundle`.
2. Запускает само приложение через `dotnet TodoApp.Web.dll`.

## Как все это запустить?

1. Убедитесь, что у вас установлен Docker и Docker Compose.
2. Откройте терминал в корне проекта.
3. Запустите сборку и запуск:

   ```bash
   docker compose up --build
   ```

4. После запуска перейдите на `http://localhost:5000/swagger`, чтобы проверить работу API.

## Частые ошибки студентов

- **Забыли права на выполнение**: Скрипт `.sh` должен иметь права на запуск. В Dockerfile мы это фиксим через `chmod +x`.
- **Неверная строка подключения**: Убедитесь, что пароли в `docker-compose.yml` совпадают с теми, что ожидает ваше приложение.
